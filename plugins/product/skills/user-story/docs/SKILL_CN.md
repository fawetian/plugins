# 用户故事助手

遵循敏捷最佳实践创建和管理用户故事的工具。

## 核心理念

**INVEST 原则**：好的用户故事应该是独立的(Independent)、可协商的(Negotiable)、有价值的(Valuable)、可估算的(Estimable)、小的(Small)、可测试的(Testable)。

## 约束

- **所有输出必须使用中文**：所有用户故事必须用中文撰写。
- **使用 Mermaid 绘制图表**：故事地图和关系图必须使用 Mermaid 语法。

## 用户故事格式

```
作为一个 [用户角色]
我想要 [完成某个目标]
以便于 [获得某种价值]
```

## 验收标准格式

```
给定 [前置条件]
当 [用户行为]
那么 [预期结果]
```

## 故事组成

### 1. 故事头部
- 故事 ID
- Epic/功能引用
- 优先级（P0/P1/P2/P3）
- 故事点（如已估算）

### 2. 故事正文
- 用户故事陈述
- 验收标准（Gherkin 格式）
- 假设和约束

### 3. 支持信息
- UI/UX 原型
- 技术说明
- 依赖关系
- 完成定义清单

---

## 工作流程

### 第一步：定义角色
定义用户画像，包括目标、痛点和行为。

### 第二步：故事映射
创建用户故事地图，展示用户旅程和故事层级。

### 第三步：编写故事
遵循 INVEST 原则编写单个故事。

### 第四步：细化
添加验收标准、估算和依赖关系。

### 第五步：优先级排序
按业务价值和依赖关系排序故事。
