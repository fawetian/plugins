# Postgres Patterns

PostgreSQL 数据库最佳实践速查手册，涵盖查询优化、Schema 设计、索引策略、行级安全（RLS）和连接池配置，基于 Supabase 最佳实践。

## 安装

```bash
/plugin install coding@fawetian-plugins
```

## 触发方式

该 skill 在以下场景下自动激活，无需显式命令：

- 编写 SQL 查询或迁移脚本
- 设计数据库 Schema
- 排查慢查询问题
- 实现行级安全策略（RLS）
- 配置连接池

在对话中提及相关操作即可触发，例如：

- `帮我优化这条 SQL 查询`
- `如何给这个表加索引`
- `如何实现 Supabase 的行级安全`

## 使用示例

**场景一：选择正确的索引类型**

询问如何为某个查询模式建索引时，skill 根据查询类型（等值查询、范围查询、JSONB 包含、全文检索、时序范围）推荐对应的索引类型（B-tree、GIN、BRIN），并提供具体 SQL。

**场景二：实现游标分页替代 OFFSET**

需要列表分页时，skill 推荐基于主键的游标分页（`WHERE id > $last_id`），说明其 O(1) 复杂度优势，并对比 OFFSET 的 O(n) 性能问题。

**场景三：诊断数据库性能问题**

skill 提供常用诊断 SQL，包括查找未建索引的外键、通过 `pg_stat_statements` 识别慢查询、检查表膨胀（dead tuples）等，辅助定位瓶颈。

## 输出

该 skill 以内联指导方式输出，直接在对话中提供：

- 索引类型速查表（查询模式 → 索引类型 → 示例 SQL）
- 数据类型选择参考（推荐类型 vs 应避免的类型）
- 常用 SQL 模式（Upsert、游标分页、队列处理、RLS 策略）
- 数据库配置模板（连接数、超时、监控扩展）
- 反模式检测 SQL（未索引外键、慢查询、表膨胀）
