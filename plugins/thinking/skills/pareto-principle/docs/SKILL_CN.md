# 帕累托法则 (80/20 法则)

## 核心理念
**聚焦关键少数，忽略琐碎多数。** 大约 80% 的产出来自 20% 的投入。找到那 20%。

## 约束
- 所有输出使用中文
- 必须量化（或估算）影响分布
- 不盲目套用 80/20 — 尽可能用数据验证
- 使用 `templates/pareto-analysis.md` 作为输出结构

## 适用场景
- 资源有限需要聚焦
- 任务清单过长需要砍 scope
- 优化性能/成本/质量时找杠杆点
- 评估投入产出比

## 不适用场景
- 所有选项同等重要（安全合规清单 — 每项都必须做）
- 需要穷尽分析不能遗漏（-> `mece`）
- 问题不是"做什么"而是"怎么想"（-> `first-principles`）

## 工作流程

### 第一步：列出所有输入项
把所有候选项/任务/因素列出来：

| # | 输入项 | 预期产出/影响 | 所需投入 |
|---|--------|-------------|---------|
| 1 | ... | ... | ... |

### 第二步：估算影响分布
为每个输入项打分或量化其贡献：
- 如果有数据 -> 用数据
- 如果没有 -> 用团队共识估算（T-shirt sizing: S/M/L/XL）

### 第三步：排序并画分布
按影响从大到小排序，计算累计百分比：

| 排名 | 输入项 | 影响占比 | 累计占比 |
|------|--------|---------|---------|
| 1 | ... | 35% | 35% |
| 2 | ... | 25% | 60% |
| 3 | ... | 15% | 75% |
| ... | ... | ... | ... |

### 第四步：画线——找到"关键少数"
在累计占比达到 ~80% 处画线：
- **线上**：关键少数（必须做）
- **线下**：琐碎多数（可以延后/砍掉/委派）

### 第五步：决策
对关键少数：
- 优先投入资源
- 设定明确的成功标准

对琐碎多数：
- 延后 / 砍掉 / 简化 / 委派
- 记录为什么不做（避免被重新提起时重复讨论）

## 产出
生成优先级分析报告。使用 `templates/pareto-analysis.md` 模板。
