# 能力圈 (Circle of Competence)

## 核心理念
**知道你不知道什么比知道你知道什么更有价值。** 你的圈子边缘是昂贵错误发生的地方。

## 约束
- 所有输出必须为中文
- 必须诚实 — 这个练习只有在知识诚实时才有效
- 必须产生可行的边界，而非含糊的承认
- 使用 `templates/competence-map.md` 作为输出结构

## 适用场景
- 做跨领域决策时
- 决定是否自己做 vs 委派/咨询
- 评估自己或团队能力
- 招聘决策
- 发现自己在信心满满地谈论不熟悉的领域

## 不适用场景
- 需要学习新领域（先用 `feynman-technique` 学）
- 需要分析具体问题（使用 `five-whys` / `mece`）
- 需要快速行动（使用 `ooda-loop`）

## 工作流程

### 第 1 步：选择评估领域
> "我要评估我在 _______ 领域的能力边界。"

### 第 2 步：绘制三环

```
┌──────────────────────────────┐
│      不知道自己不知道          │  ← 盲区（最危险）
│  ┌─────────────────────┐     │
│  │  知道自己不知道       │     │  ← 认知边界（可以学习或求助）
│  │  ┌──────────────┐   │     │
│  │  │ 知道自己知道   │   │     │  ← 核心能力圈
│  │  └──────────────┘   │     │
│  └─────────────────────┘     │
└──────────────────────────────┘
```

### 第 3 步：填充核心能力圈
列出你确实掌握的知识/技能，标准严格：
- **能教别人** = 真的懂
- **能在压力下使用** = 真的熟练
- **做过真实项目** = 有实战经验

| 我确实掌握的 | 证据（项目/经验/年限） | 置信度 |
|-------------|---------------------|--------|
| ... | ... | 高/中 |

### 第 4 步：填充认知边界
列出你知道自己不懂的：

| 我知道我不懂的 | 缺什么 | 重要性 | 行动 |
|---------------|--------|--------|------|
| ... | 经验/知识/训练 | 高/中/低 | 学习/委派/咨询 |

### 第 5 步：探索盲区
最难的部分——你不知道你不知道什么。启发方法：
- 问该领域的专家："我可能忽略了什么？"
- 回忆最近的意外（意外 = 盲区的信号）
- 检查过去自信但事后证明错误的判断

| 潜在盲区 | 发现方式 | 风险 |
|---------|---------|------|
| ... | 专家反馈/事后回顾/意外事件 | ... |

### 第 6 步：制定决策规则

| 决策类型 | 在能力圈内？ | 策略 |
|---------|------------|------|
| 我擅长的领域 | 圈内 | 自主决策 |
| 有一定了解 | 边界 | 做决策但寻求评审 |
| 不太了解 | 圈外 | 委派给专家，自己学习 |
| 完全未知 | 盲区 | 先承认无知，再找导师 |

## 产出
生成能力圈评估。使用 `templates/competence-map.md` 模板。
