# 稻草人加强 (Steelmanning)

## 核心理念
**击败对方论点的最强版本，而不是最弱的。** 如果你只能打败稻草人，你没有赢得任何东西。

## 约束
- 所有输出必须为中文
- 稻草人版必须真正强于原始版本 — 不是伪装的稻草人
- 必须用"互换测试"检查 — 对方会认同这代表他们的最佳论点吗？
- 使用 `templates/steelman-analysis.md` 作为输出结构

## 适用场景
- 立场对立的讨论（技术选型、设计评审）
- PR 评审中对别人的方案有质疑
- 听到一个你本能反对的观点
- 需要做出二选一的决策

## 不适用场景
- 对方的论点有事实错误（先纠正事实，再稻草人加强）
- 需要的是分析而非辩论（使用 `mece` / `first-principles`）
- 时间紧迫无法深入讨论（使用 `ooda-loop`）

## 工作流程

### 第 1 步：陈述对方的原始论点
如实记录对方的论点，不添加不删减：
> "对方认为 _______ 。"
> "对方的主要理由是 _______ 。"

### 第 2 步：找出对方论点的核心洞察
每个有人认真持有的观点，背后都有一个合理的核心。找到它：
> "这个观点背后的核心洞察是 _______ 。"

引导问题：
- 在什么情境下这个观点是 100% 正确的？
- 对方可能看到了什么我没看到的？
- 如果对方是 X 领域的 10 年专家，他的经验支撑了什么？

### 第 3 步：加强——构建最强版本
在对方的基础上，把论点加强到最强版本：
- 补充对方可能遗漏的支持性证据
- 用更精确的语言重新表述
- 找到更好的类比或框架

> **稻草人加强版本**："_______ "

### 第 4 步：互换测试
> "如果我把这个稻草人加强版本发给对方，他们会说'是的，这就是我的意思，甚至比我说得更好'吗？"
- 如果是 → 稻草人加强合格
- 如果不是 → 回到第 3 步继续加强

### 第 5 步：在最强版本上回应
现在针对稻草人加强版本（不是原始版本）做出你的回应：

| 对方最强论点 | 我的回应 | 我承认对方对的部分 |
|-------------|---------|------------------|
| ... | ... | ... |

**关键**：先说"你说得对的是 _______"，再说"但我认为 _______"

## 产出
生成稻草人加强分析报告。使用 `templates/steelman-analysis.md` 模板。
