# Security Review

在实现认证授权、处理用户输入、操作敏感数据或创建 API 端点时，自动提供全面的安全检查清单和安全编码模式。

## 安装

```bash
/plugin install coding@fawetian-plugins
```

## 触发方式

该 skill 在以下场景下自动激活，无需显式命令：

- 实现认证或授权逻辑
- 处理用户输入或文件上传
- 创建新的 API 端点
- 操作密钥或凭证
- 实现支付功能
- 存储或传输敏感数据
- 集成第三方 API

在对话中提及相关操作即可触发，例如：

- `帮我实现 JWT 登录`
- `如何安全地处理文件上传`
- `这个 API 端点需要什么安全措施`

## 使用示例

**场景一：密钥管理审查**

编写含有 API Key 或数据库连接串的代码时，skill 立即检测硬编码密钥，提供从环境变量读取的标准模式，并给出启动时验证密钥存在性的示例代码。

**场景二：输入验证与 SQL 注入防护**

处理用户提交的数据时，skill 推荐使用 Zod（TypeScript）等 Schema 验证库，并检查是否存在 SQL 字符串拼接，提供参数化查询的替代方案。

**场景三：生产部署前的安全清单**

在准备部署前，skill 提供覆盖 17 项的完整安全检查清单，包括：密钥管理、输入验证、SQL 注入防护、XSS 防护、CSRF 保护、速率限制、HTTPS 强制、安全响应头、错误信息脱敏、日志脱敏、依赖漏洞扫描等。

## 输出

该 skill 以内联指导方式输出，直接在对话中提供：

- 各安全类别的代码示例（错误做法 vs 正确做法对比）
- 可操作的分项检查清单（含复选框）
- Supabase RLS 策略模板
- 安全测试用例示例（认证、授权、输入验证、速率限制）
- OWASP Top 10 等参考资源链接
