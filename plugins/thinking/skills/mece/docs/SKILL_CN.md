# MECE（互斥完全）

## 核心理念
**不重叠，不遗漏。** 每个元素恰好属于一个类别，且所有元素都被覆盖。

## 约束
- 所有输出使用中文
- 在定稿前必须同时验证 ME（互斥）和 CE（完全）
- 使用 `templates/mece-decomposition.md` 作为输出结构

## 适用场景
- 拆解复杂分析，确保不遗漏不重复
- 组织方案选项，让比较公平
- 做团队分工，确保职责清晰
- 根因分析前的问题空间划分

## 不适用场景
- 问题需要创造性跳跃（→ `first-principles`）
- 需要追溯因果链（→ `five-whys`）
- 问题本质模糊、定义不清（先用 `feynman-technique` 厘清）

## 工作流程

### 第一步：定义分解对象
明确要拆分的"整体"是什么。写成一句话：
> "我要将 _______ 拆分为互斥完全的子类。"

### 第二步：选择分解维度
常用维度（选一个最贴切的）：
- **按流程**：输入 → 处理 → 输出
- **按主体**：用户 / 系统 / 第三方
- **按时间**：过去 / 现在 / 未来
- **按 Yes/No**：是 X / 不是 X
- **按量级**：大 / 中 / 小

### 第三步：列出子类

| 子类 | 定义（边界条件） | 示例 |
|------|------------------|------|
| A | ... | ... |
| B | ... | ... |
| C | ... | ... |

### 第四步：验证互斥性 (ME)
逐对检查：任意两个子类之间是否有元素同时属于两者？
- **检查方法**：找一个边界案例，问"它属于 A 还是 B？"

### 第五步：验证完全性 (CE)
问："有没有元素不属于任何子类？"
- **检查方法**：列举 3 个极端案例，确认都能归入某一类

### 第六步：输出结构化结果
```
整体：{分解对象}
维度：{选用的维度}
├── 子类 A：{定义}
├── 子类 B：{定义}
├── 子类 C：{定义}
└── (验证：ME ✓ / CE ✓)
```

## 产出
生成分析文档，包含上述 6 步的完整产物。使用 `templates/mece-decomposition.md` 模板。
